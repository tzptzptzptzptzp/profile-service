# コミット

Webアプリケーション開発における効率的で整理されたコミットを実現するために、以下のガイドラインに従います。

## 基本方針

- **小さな単位でコミットを行う**:
  - 変更はできるだけ小さく、一つの機能や修正に関連づけることで、履歴の追跡と管理が容易になる

- **WIPコミットを作成しない**:
  - 進行中のワーク（Work In Progress）はコミットせず、開発の完了または部分的に機能する状態でコミットする

## コミットメッセージ

- **日本語を使用する**:
  - 英語を使用すると直感的に理解しづらいため日本語を使用する

- **内容が理解しやすいコミットメッセージにする**:
  - 修正した内容や目的がすぐに分かるように、簡潔で明確なメッセージを心がける

- **Gitmojiを使用する**:
  - メッセージに視覚的な情報を追加するためにGitmojiを利用し、コミットの種類や意図を一目で把握できるようする

- **コミット時のチェック**:
- コミット時にHuskyにより型チェック、Lint、フォーマットが自動的に行われる  
また、コミットメッセージの内容もチェックを行い、ガイドラインに基づいていることを確認する

### コミットメッセージのサンプル

```
✨ ボタンコンポーネントを作成
🐛 ユーザー情報が取得できない問題を修正
📝 READMEを更新
```

以上のガイドラインに従うことで、コミット履歴が整理され、チームのコラボレーションが円滑に進みます。
