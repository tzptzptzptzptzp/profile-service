# 画面設計書

このドキュメントは、Webアプリケーションの整理された画面設計を実現し、開発プロセスの円滑化を図るためのガイドラインを示しています。

## サービスユーザー

### 新規登録ページ

- **ページ名**：
  - 新規登録ページ
- **URL**：
  - `/create-instance`
- **認証**：
  - 不要
- **概要**：
  - サービスユーザーの登録用ページ
- **UI要素**：
  - 入力フィールド（ユーザー名、メールアドレス、パスワード）
  - 登録ボタン
- **ユーザーフロー**：
  - 情報入力後、登録ボタンで確認メールを送信し、ログインページに遷移される
- **レスポンシブ対応**：
  - フォームレイアウトがスクリーンサイズに応じて変化

### ログインページ

- **ページ名**：
  - ログインページ
- **URL**：
  - `/access-instance`
- **認証**：
  - 不要
- **概要**：
  - サービスユーザーのログイン用ページ
- **UI要素**：
  - 入力フィールド（メールアドレス、パスワード）
  - ログインボタン
  - パスワードリセットリンク
- **ユーザーフロー**：
  - 認証情報を入力しログインボタンを押すとダッシュボードに遷移される
- **レスポンシブ対応**：
  - フォームレイアウトがスクリーンサイズに応じて変化

### パスワードリセットページ

- **ページ名**：
  - パスワードリセットページ
- **URL**：
  - `/forgot-instance`
- **認証**：
  - 不要
- **概要**：
  - サービスユーザーのパスワードリセット用ページ
- **UI要素**：
  - 入力フィールド（メールアドレス）
  - 送信ボタン
- **ユーザーフロー**：
  - メールアドレス送信後、確認メールが送られ、パスワード変更ページに遷移される
- **レスポンシブ対応**：
  - フォームレイアウトがスクリーンサイズに応じて変化

### ダッシュボード

- **ページ名**：
  - ダッシュボード
- **URL**：
  - `/manage-instance`
- **認証**：
  - 必要
- **概要**：
  - ユーザーデータの編集ができる
- **UI要素**：
  - ナビゲーションメニュー
  - ユーザーデータ編集フォーム
  - 統計表示
  - 獲得バッジ表示
- **ユーザーフロー**：
  - ログイン後に遷移
  - GitHubアカウントの連携
  - ユーザープロフィールの編集
  - メニューから各セクションに移動
- **レスポンシブ対応**：
  - フォームレイアウトがスクリーンサイズに応じて変化
  - ナビゲーションがハンバーガーメニューに変化

### パスワード変更ページ

- **ページ名**：
  - パスワード変更ページ
- **URL**：
  - `/reset-instance`
- **認証**：
  - 必要
- **概要**：
  - サービスユーザーのパスワード変更用ページ
- **UI要素**：
  - 入力フィールド（現在のパスワード、新しいパスワード）
  - 変更ボタン
- **ユーザーフロー**：
  - 新しいパスワードを入力し変更ボタンを押すとパスワードが更新される
- **レスポンシブ対応**：
  - フォームレイアウトがスクリーンサイズに応じて変化

### プロフィールページ

- **ページ名**：
  - プロフィールページ
- **URL**：
  - `/[userId]`
- **認証**：
  - 不要
- **概要**：
  - サービスにおけるメインページ
  - ユーザーが設定したプロフィールを表示
  - 指定したGitHubアカウントの情報を取得し、**"イイ感じ"**に加工して表示
- **UI要素**：
  - プロフィール情報
  - GitHubデータ
  - 編集ボタン
- **ユーザーフロー**：
  - ユーザーIDに基づいてページにアクセス
  - 自身のプロフィールページの場合は直接編集ができる
- **レスポンシブ対応**：
  - 情報カードのレイアウトが動的に変化

## 管理者

### ログインページ

- **ページ名**：
  - 管理者ログインページ
- **URL**：
  - `/admin/access-instance`
- **認証**：
  - 不要
- **概要**：
  - 管理者ユーザーのログイン用ページ
- **UI要素**：
  - 入力フィールド（メールアドレス、パスワード）
  - ログインボタン
- **ユーザーフロー**：
  - 認証後、管理者ダッシュボードへ遷移される
- **レスポンシブ対応**：
  - PCのみのレイアウトとする

### ダッシュボード

- **ページ名**：
  - 管理者ダッシュボード
- **URL**：
  - `/admin/manage-instance`
- **認証**：
  - 必要
- **概要**：
  - ログイン後に遷移するページ
  - ユーザー数、アクティブユーザー数などの確認ができる
  - アクセス数は`Google Analytics`で確認
- **UI要素**：
  - ナビゲーション
  - データグラフ
  - ユーザー管理セクション
- **ユーザーフロー**：
  - 管理者権限に基づく機能アクセス
  - 各データに基づいて操作可能
- **レスポンシブ対応**：
  - PCのみのレイアウトとする

## 共通

### 404 エラーページ

- **ページ名**：
  - 404 エラーページ
- **URL**：
  - `/404`
- **認証**：
  - 不要
- **概要**：
  - リソースが見つからない場合に表示されるページ
- **UI要素**：
  - エラーメッセージ
  - ホームへのリンク
- **ユーザーフロー**：
  - 存在しないページにアクセスした場合にこのページに遷移する
- **レスポンシブ対応**：
  - 全画面サイズでの表示に対応

### 500 エラーページ

- **ページ名**：
  - 500 エラーページ
- **URL**：
  - `/500`
- **認証**：
  - 不要
- **概要**：
  - サーバーで何らかのエラーが発生した場合に表示されるページ
- **UI要素**：
  - エラーメッセージ
  - サポート問い合わせリンク
- **ユーザーフロー**：
  - エラーが発生した場合にこのページに遷移する
- **レスポンシブ対応**：
  - 全画面サイズでの表示に対応

## 作成しないページ

### アプリケーショントップページ

- サービスサイトはアプリケーション内で作成せず、サブドメインで作成する

### 管理者新規登録ページ

- アプリケーション内で管理者ユーザーの作成はできない
