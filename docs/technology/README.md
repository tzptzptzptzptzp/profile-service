# 技術選定

このプロジェクトでは、フロントエンドとバックエンドの両方を1つのNext.jsアプリケーションで完結させることで、統一された開発環境と効率的なデプロイを実現します。

## 全体

### 言語

- **JavaScript**
  - 広く普及しており、フロントエンドとバックエンドの両方で使用できるため

- **TypeScript**
  - 型安全性を提供し、バグを未然に防ぐことで開発の安定性を向上させるため

### ライブラリ

- **React**
  - コンポーネントベースの開発スタイルが効率的で、再利用性が高いため

#### フレームワーク

- **Next.js**
  - サーバーサイドレンダリングをサポートし、SEOとパフォーマンスを向上させるため

#### 日付操作

- **date-fns**
  - 軽量で直感的な日付操作が可能で、パフォーマンスが優れているため

#### ツール

- **eslint**
  - コードの一貫性を維持し、潜在的な問題を早期に発見するため

- **husky**
  - Gitフックを使ってコマンドを実行し、コード品質を守るため

- **prettier**
  - コードフォーマットを自動化し、一貫したスタイルを保つため

## フロントエンド

### ライブラリ

#### 状態管理

- **Zustand**
  - シンプルでフレキシブルな状態管理が可能で、導入が容易なため

#### フォームとバリデーション

- **React Hook Form**
  - 高パフォーマンスかつ簡単にフォームのバリデーションを実装できるため

- **Zod**
  - スキーマベースで型の安全なバリデーションを提供するため

#### データフェッチング

- **TanStack Query**
  - データフェッチングとキャッシュ管理をシンプルにし、アプリのパフォーマンスを向上させるため

#### スタイリング

- **sass**
  - 変数やネストなどの機能を提供し、CSSを拡張するため

- **tailwindcss**
  - ユーティリティファーストなデザインアプローチを提供し、迅速なスタイリングを可能にするため

- **tailwind-merge**
  - Tailwind CSSのクラス競合を解決し、コードを簡潔に保つため

- **clsx**
  - 複数のCSSクラスを簡単に切り替えることで、ダイナミックなスタイリングを可能にするため

#### テスト

- **jest**
  - 豊富な機能を備えたJavaScript向けのテスティングフレームワークで、ユニットテストや統合テストを支援するため

- **cypress**
  - エンドツーエンドテストを簡単に行い、ユーザー視点でのテストが可能になるため

## バックエンド

#### ORM

- **Prisma**
  - 型安全で強力なデータベース操作が可能な次世代ORMとして選定

#### データベース

- **Supabase**
  - オープンソースでリアルタイムデータベースの機能を提供し、スケーラブルなバックエンドを構築するため

### ライブラリ

#### 環境変数

- **dotenv**
  - 環境変数の読み込みを容易にし、設定の管理を簡略化するため

#### テスト

- **jest**
  - フロントエンドと共通のテスティングフレームワークを使用することで、開発の一貫性と効率を向上させるため

この技術選定とルールに従うことで、品質と効率の両立を図り、プロジェクトの成功を目指します。継続的な改善を心がけ、チーム全体で協力し、より良いソリューションを提供します。
