# テスト

## フロントエンド

フロントエンドのテストには、JestおよびCypressを使用します。

- **Jest**: ユニットテストとスナップショットテストに使用します。Reactコンポーネントやユーティリティ関数の動作を確認します。

- **Cypress**: エンドツーエンド（E2E）テストに使用します。ユーザー操作をシミュレートし、アプリケーションの全体的な動作を確認します。

### テスト実行コマンド

- Jestテストを実行: `npm run test`
- Cypressテストを実行: `npm run cypress:open` または `npm run cypress:run`

## バックエンド

バックエンドのテストには、Jestを使用します。

- **Jest**: APIエンドポイントやビジネスロジックのユニットテストを行います。

### テスト実行コマンド

- Jestテストを実行: `npm run test`

## CI（継続的インテグレーション）

継続的インテグレーション（CI）は、ソフトウェア開発において頻繁にコードを統合し、自動化されたビルドとテストを実行するプロセスです。これにより、コードベースに問題がある場合に早期に検出できます。

### GitHub ActionsによるCI設定

GitHub Actionsを使用して、プルリクエスト（PR）が作成された際に自動的にテストを実行します。以下は、CI/CDパイプラインを設定するためのワークフローファイルの例です。

### CIの利点

- **即時フィードバック**: コードの品質や問題について迅速にフィードバックを得ることができます。
- **チーム全体の効率向上**: 自動化されたテストにより、開発チームは手動での確認作業から解放されます。
- **高品質なコードベース**: 継続的なテストにより、コードの安定性と品質を維持できます。

この設定により、プロジェクト内のコードが新しい変更に対して安全かどうかを素早く確認できます。
