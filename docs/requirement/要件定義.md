# 要件定義

## 目的

- エンジニアが最小限の時間で魅力的なプロフィールを作成することができるようにすること

## 要求定義

- 面倒な設定不要をすることなく GitHub の情報を元に "イイ感じ" にプロフィールが作成できる
- 職務経歴書などのようにエンジニアとしてのスキルや経歴を管理し公開できる
- 表示する項目をユーザーが設定できる
- パスワードによる閲覧制限を設定できる

## ペルソナ定義

- エンジニア・プロダクトマネージャー
- 性別問わず
- 20 代〜30 代

## 機能要件

- ユーザー登録・ログインができる
- ログアウトができブラウザ上からログイン情報が削除される
- GitHub アカウントを登録することで 以下の内容が表示される
  - アイコン
  - contributions に応じたランク
  - 月間、週間の contributions 数
  - 活動内容に応じたバッジ
  - ユーザー名などは GitHub で設定しているものが初期値となる
- スキルの設定・公開ができる
- プロジェクトの設定・公開ができる
- スマートフォンでも適切に表示される

## 非機能要件

- 高速な応答時間を保証する
- ユーザー数の増加に対応可能である
- セキュリティプロトコル（HTTPS、OAuth 2.0 など）を使用する
- サービスの稼働率を 99.9%以上にする
- コードはモジュール化され、容易に保守できる
- 直感的でレスポンシブなユーザーインターフェースを提供する
- 多言語対応を視野に入れる
- 定期的なデータバックアップを行う

## プロジェクト管理

- Issue ベースで機能開発を行う
- Issue に対応したブランチを作成し開発を行う
- Pull Request を作成し Copilo Review を行う
- 継続的インテグレーション（CI）ツールを用いて自動テストを実行する
- 継続的デプロイメント（CD）を導入し、自動的にステージング及びプロダクション環境にデプロイされるようにする
- 工数管理は Issue に予定工数、Pull Request に実働工数を入力する
- ドキュメント管理をリポジトリ内で行う
